# Story 10.1 — Backend + Frontend — Streak e Gamificação Básica

## Status: Draft

## Story

**Como** usuário do EduTrack,
**Quero** ver minha sequência de dias consecutivos de estudo (streak),
**Para que** me sinta motivado a manter a consistência e não quebrar a sequência.

---

## Contexto

Streak é um dos mecanismos de engajamento mais efetivos em apps de estudo (Duolingo, Anki). Um "dia de estudo" é contabilizado quando o usuário tem pelo menos 1 sessão com `status: 'completed'` naquele dia. O streak é calculado a partir de ontem para trás (hoje pode ainda não ter sessão, mas a sequência não deve quebrar).

**Escopo desta story:**
- Cálculo de streak no backend via endpoint dedicado
- Exibição do streak no Dashboard (Home) com ícone de fogo e número
- Exibição de melhor streak histórico
- Mensagens motivacionais baseadas no streak atual

**Fora de escopo:**
- Notificações de streak prestes a quebrar (Epic futuro)
- Achievements/badges (Epic futuro)
- Streak freeze (Epic futuro)

## Dev Notes

### Algoritmo de streak (Ruby)

```ruby
def streak
  # Datas únicas com sessão completada, ordenadas desc
  study_dates = StudySession
    .joins(topic: :subject)
    .where(subjects: { user_id: current_user.id })
    .where(status: 'completed')
    .where('questions_done > 0')
    .pluck("DISTINCT DATE(scheduled_date)")
    .map(&:to_date)
    .sort
    .reverse

  today = Date.today
  studied_today = study_dates.first == today
  last_study_date = study_dates.first

  # current_streak: contar a partir de hoje (ou ontem se hoje não estudou)
  current_streak = 0
  check_date = studied_today ? today : today - 1
  study_set = study_dates.to_set

  while study_set.include?(check_date)
    current_streak += 1
    check_date -= 1
  end

  # longest_streak: percorrer todas as datas
  longest = 0
  current = 0
  prev_date = nil

  study_dates.reverse.each do |date|
    if prev_date.nil? || date == prev_date + 1
      current += 1
      longest = [longest, current].max
    else
      current = 1
    end
    prev_date = date
  end

  render json: {
    current_streak: current_streak,
    longest_streak: longest,
    last_study_date: last_study_date&.iso8601,
    studied_today: studied_today
  }
end
```

### Nota sobre "dia de estudo"

Um dia conta se tem `questions_done > 0` — garante que sessões vazias (sem questões respondidas) não contam como estudo real. Este critério pode ser relaxado futuramente para `status: 'completed'` sem filtro de questões.
