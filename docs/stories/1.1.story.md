# Story 1.1 — Setup Monorepo + Rails API

## Status: Done

## Story

**Como** desenvolvedor do projeto,
**Quero** ter o monorepo estruturado com o backend Rails configurado,
**Para que** o time possa começar o desenvolvimento sobre uma base sólida e padronizada.

---

## Contexto

Este é o projeto **EduTrack** — aplicativo PWA para gestão de estudos para vestibular/concurso.

**Stack definida:**
- Backend: Ruby on Rails 7+ (API mode)
- Frontend: Angular 17+ PWA (servido pelo Rails em `public/`)
- Banco: PostgreSQL
- Auth: Devise + Devise-JWT
- Deploy: Railway (serviço único)

Esta story cobre apenas o setup do monorepo e da aplicação Rails com banco configurado. Angular e Auth são stories separadas (1.2 e 1.3).

### Configuração do banco (`config/database.yml`)

```yaml
default: &default
  adapter: postgresql
  encoding: unicode
  pool: <%= ENV.fetch("RAILS_MAX_THREADS") { 5 } %>
  username: <%= ENV['DATABASE_USERNAME'] %>
  password: <%= ENV['DATABASE_PASSWORD'] %>
  host: <%= ENV.fetch('DATABASE_HOST') { 'localhost' } %>

development:
  <<: *default
  database: edutrack_development

test:
  <<: *default
  database: edutrack_test

production:
  <<: *default
  url: <%= ENV['DATABASE_URL'] %>
```

### Variáveis de ambiente necessárias (`.env`)

```
DATABASE_USERNAME=postgres
DATABASE_PASSWORD=postgres
DATABASE_HOST=localhost
```
