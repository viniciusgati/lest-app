# Story 1.5 — Suite de Testes — RSpec + Jest com Relatório Minimalista

## Status: Ready for Review

## Story

**Como** desenvolvedor do EduTrack,
**Quero** ter uma suite de testes funcional no backend e frontend com relatório enxuto,
**Para que** eu possa validar o sistema rapidamente com o mínimo de contexto necessário para análise.

## Contexto

Última story do Epic 1. Backend Rails e frontend Angular estão configurados com auth e layout. Esta story instala e configura os frameworks de teste com formatadores customizados que exibem **apenas falhas e contagem total** — suprimindo testes que passaram para reduzir o ruído no contexto de análise.

### Formato de saída esperado

```
Tests: 47 passed, 0 failed

# Quando há falhas:
Tests: 44 passed, 3 failed

FAILED:
  [backend] spec/requests/health_spec.rb:8 — expected 200, got 500
  [backend] spec/models/user_spec.rb:15 — User is invalid without name
  [frontend] src/app/core/services/auth.service.spec.ts:22 — should store token on login
```

**Dependência:** Stories 1.1 a 1.4 concluídas.

## Dev Notes

### Formatter minimalista do RSpec

```ruby
# backend/spec/support/minimal_formatter.rb
class MinimalFormatter
  RSpec::Core::Formatters.register self,
    :start, :example_failed, :dump_summary

  def initialize(output)
    @output = output
    @failures = []
  end

  def start(notification)
    # Silencioso no início
  end

  def example_failed(notification)
    example = notification.example
    @failures << {
      location: example.location,
      description: example.full_description,
      message: notification.message_lines.first&.strip
    }
  end

  def dump_summary(notification)
    passed = notification.example_count - notification.failure_count
    failed = notification.failure_count

    @output.puts "\nTests: #{passed} passed, #{failed} failed"

    if @failures.any?
      @output.puts "\nFAILED:"
      @failures.each do |f|
        @output.puts "  [#{f[:location]}] #{f[:message]}"
      end
    end
  end
end
```

    end

### Reporter minimalista do Jest

```javascript
// frontend/jest-minimal-reporter.js
class MinimalReporter {
  constructor(globalConfig, options) {
    this._globalConfig = globalConfig;
    this._failures = [];
  }

  onRunComplete(contexts, results) {
    const passed = results.numPassedTests;
    const failed = results.numFailedTests;

    console.log(`\nTests: ${passed} passed, ${failed} failed`);

    if (failed > 0) {
      console.log('\nFAILED:');
      results.testResults.forEach(suite => {
        suite.testResults
          .filter(t => t.status === 'failed')
          .forEach(test => {
            const file = suite.testFilePath.replace(process.cwd(), '');
            const msg = test.failureMessages[0]?.split('\n')[0] ?? '';
            console.log(`  [${file}] ${test.fullName} — ${msg}`);
          });
      });
    }
  }
}

module.exports = MinimalReporter;
```

### Script `test.sh` (raiz do monorepo)

```bash
#!/bin/bash
set -e

BACKEND_EXIT=0
FRONTEND_EXIT=0

echo "=== Backend (RSpec) ==="
cd backend && bundle exec rspec || BACKEND_EXIT=$?
cd ..

echo ""
echo "=== Frontend (Jest) ==="
cd frontend && npm test || FRONTEND_EXIT=$?
cd ..

echo ""
if [ $BACKEND_EXIT -eq 0 ] && [ $FRONTEND_EXIT -eq 0 ]; then
  echo "✅ All tests passed"
  exit 0
else
  echo "❌ Tests failed (backend: $BACKEND_EXIT, frontend: $FRONTEND_EXIT)"
  exit 1
fi
```
