# Story 4.3 — Frontend — Registrar Resultado + Sugestão SM-2

## Status: Ready for Review

## Story

**Como** usuário do EduTrack,
**Quero** registrar o resultado de uma sessão de estudo e ver a próxima revisão sugerida,
**Para que** meu desempenho seja registrado e o sistema agende automaticamente minha próxima revisão.

---

## Contexto

Continuação da Story 4.2. Calendário funcionando. Esta story implementa o fluxo de conclusão de uma sessão: formulário de resultado → chamada ao backend → exibição da sugestão SM-2 com opção de aceitar ou ajustar a data.

**Dependência:** Story 4.2 concluída.

## Dev Notes

### Formulário de resultado

```typescript
resultForm = this.fb.group({
  actual_minutes: [null, [Validators.required, Validators.min(1)]],
  questions_done: [0, [Validators.required, Validators.min(0)]],
  questions_correct: [0, [Validators.required, Validators.min(0)]]
}, { validators: correctsLteDoneValidator });

function correctsLteDoneValidator(group: AbstractControl) {
  const done = group.get('questions_done')?.value ?? 0;
  const correct = group.get('questions_correct')?.value ?? 0;
  return correct > done ? { correctsExceedDone: true } : null;
}
```

### Fluxo de steps no dialog

```
Step 1: Formulário de resultado
  → [Submeter] → loading → request para /complete

Step 2: Sugestão SM-2
  ┌────────────────────────────────┐
  │ ✅ Sessão registrada!          │
  │                                │
  │ Próxima revisão sugerida:      │
  │ 📅 16 de março de 2026 (6 dias)│
  │                                │
  │ [Aceitar]  [Ajustar data]      │
  └────────────────────────────────┘

Step 3 (opcional): Ajustar data
  → p-datePicker
  → [Confirmar nova data]
```

### Evento emitido para o calendário

```typescript
// Evento emitido para AgendaComponent
export interface SessionCompletedEvent {

  nextSession: StudySession;
}
```

### Template do card SM-2 (Step 2)

```html
<div class="text-center p-4">
  <i class="pi pi-check-circle text-green-500 text-5xl mb-3 block"></i>
  <h3 class="mb-1">Sessão registrada!</h3>
  <p class="text-color-secondary mb-4">
    {{ questions_correct }} / {{ questions_done }} questões certas
    ({{ percentage }}%)
  </p>

  <p-card class="mb-4">
    <div class="text-center">
      <div class="text-lg font-bold mb-1">📅 Próxima revisão sugerida</div>
      <div class="text-2xl font-bold text-primary">{{ nextReviewLabel }}</div>
      <div class="text-color-secondary">em {{ interval }} {{ interval === 1 ? 'dia' : 'dias' }}</div>
    </div>
  </p-card>

  @if (!adjustingDate) {
    <div class="flex gap-2 justify-content-center">
      <p-button label="Aceitar" icon="pi pi-check" (onClick)="accept()" />
      <p-button label="Ajustar data" icon="pi pi-calendar" [outlined]="true" (onClick)="adjustingDate = true" />
    </div>
  } @else {
    <p-datePicker [(ngModel)]="adjustedDate" [inline]="true" [minDate]="tomorrow" />
    <p-button label="Confirmar nova data" icon="pi pi-check" class="mt-3" (onClick)="confirmAdjusted()" />
  }
</div>
```
