## Dev Notes

### Versões
- Angular CLI: 17+
- Node: 18+
- PrimeNG: 17+ (compatível com Angular 17)

### Comando de criação do projeto

```bash
cd /path/to/monorepo
ng new frontend --routing --style=scss --ssr=false
cd frontend
ng add @angular/pwa
npm install primeng @primeng/themes primeicons
```

### Configuração do PrimeNG em `app.config.ts`

```typescript
import { ApplicationConfig } from '@angular/core';
import { provideRouter } from '@angular/router';
import { provideAnimationsAsync } from '@angular/platform-browser/animations/async';
import { providePrimeNG } from 'primeng/config';
import Aura from '@primeng/themes/aura';
import { routes } from './app.routes';

export const appConfig: ApplicationConfig = {
  providers: [
    provideRouter(routes),
    provideAnimationsAsync(),
    providePrimeNG({
      theme: {
        preset: Aura,
        options: { darkModeSelector: '.app-dark' }
      }
    })
  ]
};
```

### `styles.scss` base

> **Nota PrimeNG 17+ com Aura:** O tema é injetado programaticamente via `providePrimeNG` em `app.config.ts`. NÃO importar `primeng/resources/themes/*.css` (legado v16). Importar apenas primeicons.

```scss
/* Apenas primeicons — tema Aura é injetado via providePrimeNG() */
@import "primeicons/primeicons.css";

* {
  box-sizing: border-box;
  margin: 0;
  padding: 0;
}

body {
  font-family: var(--font-family);
  background-color: var(--surface-ground);
  color: var(--text-color);
}
```

### `angular.json` — outputPath

```json
"build": {
  "options": {
    "outputPath": "../backend/public",
    ...
  }
}
```

### Catch-all no Rails (`config/routes.rb`)

```ruby
Rails.application.routes.draw do
  get '/health', to: 'health#index'

  # API routes virão aqui (namespaced /api/v1)

  # Catch-all: delega para Angular (deve ser o ÚLTIMO)
  get '*path',
      to: 'application#fallback_index_html',
      constraints: ->(req) { !req.xhr? && req.format.html? }
end
```

### `ApplicationController` com fallback

```ruby
class ApplicationController < ActionController::API
  def fallback_index_html
    render file: Rails.root.join('public', 'index.html'), layout: false
  end
end
```

### Estrutura de rotas Angular (`app.routes.ts`)

```typescript
export const routes: Routes = [
  { path: '', loadComponent: () => import('./features/home/home.component').then(m => m.HomeComponent) },
  { path: 'subjects', loadComponent: () => import('./features/subjects/subjects.component').then(m => m.SubjectsComponent) },
  { path: 'agenda', loadComponent: () => import('./features/agenda/agenda.component').then(m => m.AgendaComponent) },
  { path: 'goals', loadComponent: () => import('./features/goals/goals.component').then(m => m.GoalsComponent) },
  { path: 'auth/login', loadComponent: () => import('./features/auth/login/login.component').then(m => m.LoginComponent) },
  { path: '**', redirectTo: '' }
];
```

### Importante: API calls durante desenvolvimento

Durante o desenvolvimento local, o Angular em `ng serve` (porta 4200) precisa fazer proxy para o Rails (porta 3000). Criar `frontend/proxy.conf.json`:

```json
{
  "/api": {
    "target": "http://localhost:3000",
    "secure": false
  },
  "/health": {
    "target": "http://localhost:3000",
    "secure": false
  }
}
```

E configurar em `angular.json`:
```json
"serve": {
  "options": {
    "proxyConfig": "proxy.conf.json"
  }
}
```
