# Story 8.3 — Backend — Soft Delete em Matérias e Temas

## Status: Draft

## Story

**Como** usuário do EduTrack,
**Quero** que ao deletar uma matéria ou tema eles sejam marcados como excluídos (não destruídos permanentemente),
**Para que** meu histórico de sessões de estudo seja preservado e eu não perca dados importantes acidentalmente.

---

## Contexto

Atualmente `DELETE /subjects/:id` destrói permanentemente o subject e, por cascade, todos os topics e study_sessions associados. Isso é destrutivo e irreversível. Com a gem `discard` (preferida ao `paranoia` por ser mais simples e não monkey-patching do ActiveRecord), adicionamos uma coluna `discarded_at` e filtramos registros descartados automaticamente via default scope.

**Impacto:**
- `Subject` com soft delete: ao excluir, `discarded_at` é preenchido, topics e sessions permanecem no banco
- `Topic` com soft delete: mesmo comportamento
- `StudySession` NÃO precisa de soft delete (ela tem integridade via `topic_id`)
- Métricas: queries existentes precisam ignorar subjects/topics descartados

## Dev Notes

### Gemfile

```ruby
gem 'discard', '~> 1.4'
```

### Subject model

```ruby
class Subject < ApplicationRecord
  include Discard::Model
  default_scope { kept }

  belongs_to :user
  has_many :topics, dependent: :destroy  # cascade discard manual no controller
end
```

### Topic model

```ruby
class Topic < ApplicationRecord
  include Discard::Model
  default_scope { kept }

  belongs_to :subject
  has_many :study_sessions, dependent: :nullify  # sessions ficam no banco
end
```

### SubjectsController#destroy

```ruby
def destroy
  ActiveRecord::Base.transaction do
    @subject.topics.discard_all  # cascade soft delete
    @subject.discard
  end
  head :no_content
end
```

### Nota sobre `discard_all`

`discard_all` é um scope method da gem discard que seta `discarded_at` em todos os registros do scope. É equivalente a `update_all(discarded_at: Time.current)`.

### Nota sobre default_scope

Com `default_scope { kept }`, todas as queries automáticas filtrarão registros descartados. Para acessar registros descartados (admin/debug): usar `Subject.with_discarded` ou `Subject.discarded`.

### Impacto em StudySessions

`StudySession.joins(topic: :subject)` ainda funcionará porque as sessions não têm soft delete. As sessions ficam "órfãs" de topics descartados mas permanecem no banco para preservar histórico. As métricas já filtram por `subject_id` do usuário, então topics descartados (via subjects descartados) não aparecerão.
