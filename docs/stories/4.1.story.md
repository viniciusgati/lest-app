# Story 4.1 — Backend — Sessões de Estudo

## Status: Draft

## Story

**Como** usuário do EduTrack,
**Quero** que o backend gerencie minhas sessões de estudo,
**Para que** eu possa agendar revisões, registrar resultados e ter meu desempenho calculado automaticamente.

---

## Contexto

Primeira story do Epic 4. Esta story cria o model `StudySession` — o coração do sistema. Cada sessão vincula um tema a uma data, registra o resultado (questões + acertos) e alimenta as métricas e o algoritmo SM-2.

Ao registrar o resultado de uma sessão, o backend:
1. Atualiza `status` de `scheduled` para `completed`
2. Recalcula SM-2 do topic (`ease_factor`, `interval`, `next_review`)
3. Cria automaticamente a próxima sessão de revisão baseada no SM-2

**Dependência:** Epics 1, 2 e 3 completos.

## Acceptance Criteria

### CRUD básico
- [ ] **AC1** — `GET /api/v1/study_sessions` retorna sessões do usuário (filtros: `?status=`, `?date_from=`, `?date_to=`)
- [ ] **AC2** — `POST /api/v1/study_sessions` agenda nova sessão (status: `scheduled`)
- [ ] **AC3** — `PUT /api/v1/study_sessions/:id` atualiza dados de uma sessão agendada
- [ ] **AC4** — `DELETE /api/v1/study_sessions/:id` remove sessão agendada
- [ ] **AC5** — Usuário não acessa sessões de outro usuário (404)

### Registro de resultado
- [ ] **AC6** — `PUT /api/v1/study_sessions/:id/complete` registra resultado: `actual_minutes`, `questions_done`, `questions_correct`
- [ ] **AC7** — Ao completar, `status` muda para `completed`
- [ ] **AC8** — Ao completar, SM-2 do topic é recalculado e salvo (`ease_factor`, `interval`, `next_review`)
- [ ] **AC9** — Ao completar, nova sessão futura é criada automaticamente com a data do SM-2 (status: `scheduled`, `auto_generated: true`)
- [ ] **AC10** — `questions_correct` não pode ser maior que `questions_done`

### Status automático
- [ ] **AC11** — Sessões com `scheduled_date < hoje` e `status: scheduled` são retornadas com `status: late` na resposta JSON (calculado, não persistido)

### Validações
- [ ] **AC12** — `scheduled_date` obrigatório; `topic_id` obrigatório e deve pertencer ao usuário
- [ ] **AC13** — `expected_minutes` >= 1; `questions_done` >= 0; `questions_correct` >= 0

### Testes
- [ ] **AC14** — RSpec cobre todos os endpoints, cálculo SM-2 e criação automática da próxima sessão

## Dev Notes

### Model StudySession

```ruby
# app/models/study_session.rb
class StudySession < ApplicationRecord
  belongs_to :topic

  validates :scheduled_date, presence: true
  validates :expected_minutes, numericality: { greater_than: 0 }
  validates :questions_done, numericality: { greater_than_or_equal_to: 0 }
  validates :questions_correct, numericality: { greater_than_or_equal_to: 0 }
  validates :status, inclusion: { in: %w[scheduled completed] }
  validate :questions_correct_lte_done

  delegate :subject, to: :topic

  def late?
    status == 'scheduled' && scheduled_date < Date.today
  end

  def status_display
    late? ? 'late' : status
  end

  private

  def questions_correct_lte_done
    return unless questions_done && questions_correct
    if questions_correct > questions_done
      errors.add(:questions_correct, 'não pode ser maior que questões feitas')
    end
  end
end
```

### SM2Calculator Service

```ruby
# app/services/sm2_calculator.rb
class Sm2Calculator
  MIN_EASE = 1.3

  def self.calculate(topic, questions_done, questions_correct)
    q = quality_score(questions_done, questions_correct)

    ease = topic.ease_factor
    interval = topic.interval

    if q >= 3
      interval = case interval
                 when 1 then 6
                 when 6 then (interval * ease).round
                 else (interval * ease).round
                 end
      ease = [ease + (0.1 - (5 - q) * (0.08 + (5 - q) * 0.02)), MIN_EASE].max
    else
      interval = 1
    end

    {
      ease_factor: ease.round(2),
      interval: interval,
      next_review: Date.today + interval.days
    }
  end

  def self.quality_score(questions_done, questions_correct)
    return 0 if questions_done.zero?
    pct = (questions_correct.to_f / questions_done * 100).round
    case pct
    when 85..100 then 5
    when 75..84  then 4
    when 60..74  then 3
    when 40..59  then 2
    else              1
    end
  end
end
```

### StudySessionsController

```ruby
# app/controllers/api/v1/study_sessions_controller.rb
module Api
  module V1
    class StudySessionsController < BaseController
      before_action :set_session, only: [:update, :destroy, :complete]

      def index
        sessions = user_sessions.order(scheduled_date: :asc)
        sessions = sessions.where(status: params[:status]) if params[:status].present?
        sessions = sessions.where(scheduled_date: params[:date_from]..) if params[:date_from].present?
        sessions = sessions.where(scheduled_date: ..params[:date_to]) if params[:date_to].present?
        render json: sessions.map { |s| session_json(s) }
      end

      def create
        session = StudySession.new(session_params)
        validate_topic_ownership!(session.topic_id)
        if session.save
          render json: session_json(session), status: :created
        else
          render json: { errors: session.errors.full_messages }, status: :unprocessable_entity
        end
      end

      def update
        if @session.update(session_params.except(:topic_id))
          render json: session_json(@session)
        else
          render json: { errors: @session.errors.full_messages }, status: :unprocessable_entity
        end
      end

      def destroy
        @session.destroy
        head :no_content
      end

      def complete
        result_params = params.require(:study_session).permit(
          :actual_minutes, :questions_done, :questions_correct
        )

        sm2 = Sm2Calculator.calculate(
          @session.topic,
          result_params[:questions_done].to_i,
          result_params[:questions_correct].to_i
        )

        ActiveRecord::Base.transaction do
          @session.update!(
            result_params.merge(status: 'completed')
          )
          @session.topic.update!(sm2)

          # Cria próxima sessão automaticamente
          next_session = StudySession.create!(
            topic: @session.topic,
            scheduled_date: sm2[:next_review],
            expected_minutes: @session.expected_minutes,
            auto_generated: true
          )

          render json: {
            session: session_json(@session),
            next_session: session_json(next_session)
          }
        end
      rescue ActiveRecord::RecordInvalid => e
        render json: { errors: [e.message] }, status: :unprocessable_entity
      end

      private

      def user_sessions
        StudySession.joins(topic: :subject).where(subjects: { user_id: current_user.id })
      end

      def set_session
        @session = user_sessions.find(params[:id])
      rescue ActiveRecord::RecordNotFound
        render json: { error: 'Not found' }, status: :not_found
      end

      def validate_topic_ownership!(topic_id)
        unless current_user.subjects.joins(:topics).where(topics: { id: topic_id }).exists?
          render json: { error: 'Topic not found' }, status: :not_found
        end
      end

      def session_json(session)
        session.as_json.merge('status' => session.status_display)
      end

      def session_params
        params.require(:study_session).permit(
          :topic_id, :scheduled_date, :expected_minutes
        )
      end
    end
  end
end
```

```ruby
RSpec.describe Sm2Calculator do
  describe '.quality_score' do
    it { expect(described_class.quality_score(10, 10)).to eq(5) }  # 100%
    it { expect(described_class.quality_score(10, 8)).to eq(4) }   # 80%
    it { expect(described_class.quality_score(10, 6)).to eq(3) }   # 60%
    it { expect(described_class.quality_score(10, 4)).to eq(2) }   # 40%
    it { expect(described_class.quality_score(10, 2)).to eq(1) }   # 20%
    it { expect(described_class.quality_score(0, 0)).to eq(0) }    # sem questões
  end

  describe '.calculate' do
    let(:topic) { build(:topic, ease_factor: 2.5, interval: 1) }

    it 'avança para intervalo 6 após primeira sessão boa' do
      result = described_class.calculate(topic, 10, 9) # 90%
      expect(result[:interval]).to eq(6)
    end

    it 'reseta intervalo para 1 após sessão ruim' do
      result = described_class.calculate(topic, 10, 2) # 20%
      expect(result[:interval]).to eq(1)
    end

    it 'define next_review corretamente' do
      result = described_class.calculate(topic, 10, 9)
      expect(result[:next_review]).to eq(Date.today + 6.days)
    end
  end
end
```
