# Story 7.1 — Testes E2E — Playwright (Fluxos Críticos)

## Status: Ready for Review

## Story

**Como** desenvolvedor do EduTrack,
**Quero** uma suite de testes E2E com Playwright cobrindo os fluxos críticos da aplicação,
**Para que** possamos detectar regressões de ponta a ponta antes de cada release.

---

## Contexto

Todas as features do EduTrack estão implementadas (Épicos 1–6). A cobertura atual é feita por testes unitários (RSpec + Vitest), mas não há testes E2E. Esta story configura o Playwright como framework E2E standalone (separado do `@vitest/browser-playwright` já instalado para testes de componente) e implementa os 6 fluxos críticos da aplicação.

**Dependência:** Todos os épicos anteriores concluídos (1.1–6.2). Backend Rails rodando em `http://localhost:3000`, Frontend Angular em `http://localhost:4200`.

---

### auth.helper.ts

```typescript
import { Page } from '@playwright/test';

let emailCounter = 0;

export async function registerAndLogin(page: Page): Promise<{ email: string; password: string }> {
  emailCounter++;
  const email = `test-${Date.now()}-${emailCounter}@e2e.test`;
  const password = 'Test1234!';

  await page.goto('/auth/signup');
  await page.getByLabel('Email').fill(email);
  await page.getByLabel('Senha').fill(password);
  await page.getByRole('button', { name: /cadastrar/i }).click();
  await page.waitForURL('/');

  return { email, password };
}
```

### Padrão de teste (exemplo: subjects.spec.ts)

```typescript
import { test, expect } from '@playwright/test';
import { registerAndLogin } from '../helpers/auth.helper';

test.describe('Matérias e Temas', () => {
  test.beforeEach(async ({ page }) => {
    await registerAndLogin(page);
  });

  test('criar matéria e tema', async ({ page }) => {
    await page.goto('/subjects');
    await page.getByRole('button', { name: /nova matéria/i }).click();
    await page.getByLabel('Nome').fill('Matemática');
    await page.getByRole('button', { name: /salvar/i }).click();
    await expect(page.getByText('Matemática')).toBeVisible();

    await page.getByText('Matemática').click();
    await page.getByRole('button', { name: /novo tema/i }).click();
    await page.getByLabel('Nome').fill('Álgebra');
    await page.getByRole('button', { name: /salvar/i }).click();
    await expect(page.getByText('Álgebra')).toBeVisible();
  });
});
```

### Notas Informacionais

**N1 — AC7 cobertura parcial do estado com dados**
- `home.spec.ts` verifica estado vazio e estrutura da página. AC7 especifica "métricas carregam após ter sessões cadastradas" — o estado positivo (accuracy exibido após sessão completada) não é testado. Para cobertura completa seria necessário criar sessão, registrar resultado via SM-2 e verificar `.subject-row` com accuracy > 0. Cobertura atual é suficiente para detectar regressões de estrutura.

**N2 — Seletor `p-button[ptooltip="Sair"]` pode ser frágil em build de produção**
- Angular compila bindings como `[pTooltip]` em atributos estáticos em dev mode. Em builds de produção (--configuration=production), os atributos de diretivas podem ser removidos. Alternativa mais robusta: `page.locator('.p-toolbar-group-end button')` ou adicionar `data-testid="logout-btn"` ao template.

**N3 — `waitForTimeout(500)` em agenda.spec.ts:50**
- Anti-pattern do Playwright: delay fixo para aguardar carregamento de temas após seleção de matéria. Deveria usar `await page.waitForResponse(...)` ou `await expect(dialog.locator('#topic_id')).toBeEnabled()`. Não impacta funcionalidade — testes com retries=1 absorvem o risco.

**N4 — Variável `email` não utilizada em schedule.spec.ts:69**
- `const { email } = await registerAndLogin(page)` — `email` não é referenciado. TypeScript compila sem erro (não usa `noUnusedLocals`). Puramente cosmético.
