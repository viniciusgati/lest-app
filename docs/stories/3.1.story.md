# Story 3.1 — Backend — Metas Semanais + Configuração do Usuário

## Status: Draft

## Story

**Como** usuário do EduTrack,
**Quero** que o backend gerencie minhas metas semanais e configurações de disponibilidade,
**Para que** o sistema possa calcular meu progresso e gerar programações automáticas de estudo.

---

## Contexto

Primeira story do Epic 3. Esta story cria dois modelos:

1. **`WeeklyGoal`** — Meta do usuário para uma semana específica (horas, questões, % alvo)
2. **`UserConfig`** — Configuração persistente do usuário (dias disponíveis + estratégia de agendamento)

O `WeeklyGoal` é identificado pela `week_start` (segunda-feira da semana). O progresso real (horas estudadas, questões feitas, % alcançado) será calculado a partir das sessões de estudo em Epic 4/5 — aqui definimos apenas as metas-alvo.

**Dependência:** Epic 1 completo.

---

## Acceptance Criteria

### WeeklyGoal
- [ ] **AC1** — `GET /api/v1/weekly_goals/current` retorna meta da semana atual (cria com zeros se não existir)
- [ ] **AC2** — `PUT /api/v1/weekly_goals/current` cria ou atualiza meta da semana atual (upsert)
- [ ] **AC3** — `GET /api/v1/weekly_goals` retorna histórico de metas do usuário (ordenado por semana desc)
- [ ] **AC4** — Meta pertence ao usuário autenticado — usuários não acessam metas uns dos outros
- [ ] **AC5** — `week_start` é sempre normalizado para a segunda-feira da semana informada
- [ ] **AC6** — Validações: `target_hours >= 0`, `target_questions >= 0`, `target_percentage` entre 0 e 100

### UserConfig
- [ ] **AC7** — `GET /api/v1/user_config` retorna configuração do usuário (cria com defaults se não existir)
- [ ] **AC8** — `PUT /api/v1/user_config` atualiza configuração do usuário
- [ ] **AC9** — `available_days` aceita array com valores válidos: `["mon","tue","wed","thu","fri","sat","sun"]`
- [ ] **AC10** — `schedule_strategy` aceita apenas: `"sm2"`, `"weak_points"`, `"balanced"`
- [ ] **AC11** — Defaults: `available_days: ["mon","tue","wed","thu","fri"]`, `schedule_strategy: "sm2"`

### Geral
- [ ] **AC12** — Todos os endpoints retornam 401 sem token JWT válido
- [ ] **AC13** — Testes RSpec cobrem todos os endpoints e validações

## Dev Notes

### Model WeeklyGoal

```ruby
# app/models/weekly_goal.rb
class WeeklyGoal < ApplicationRecord
  belongs_to :user

  validates :week_start, presence: true
  validates :target_hours, numericality: { greater_than_or_equal_to: 0 }
  validates :target_questions, numericality: { greater_than_or_equal_to: 0, only_integer: true }
  validates :target_percentage, numericality: {
    greater_than_or_equal_to: 0,
    less_than_or_equal_to: 100
  }
  validates :week_start, uniqueness: { scope: :user_id }

  before_validation :normalize_week_start

  # Retorna a segunda-feira da semana que contém a data informada
  def self.week_start_for(date = Date.today)
    date.beginning_of_week(:monday)
  end

  private

  def normalize_week_start
    self.week_start = WeeklyGoal.week_start_for(week_start) if week_start.present?
  end
end
```

### Model UserConfig

```ruby
# app/models/user_config.rb
class UserConfig < ApplicationRecord
  VALID_DAYS = %w[mon tue wed thu fri sat sun].freeze
  VALID_STRATEGIES = %w[sm2 weak_points balanced].freeze

  belongs_to :user

  validates :schedule_strategy, inclusion: { in: VALID_STRATEGIES }
  validate :validate_available_days

  private

  def validate_available_days
    return if available_days.blank?
    invalid = available_days - VALID_DAYS
    errors.add(:available_days, "contém dias inválidos: #{invalid.join(', ')}") if invalid.any?
  end
end
```

### WeeklyGoalsController

```ruby
# app/controllers/api/v1/weekly_goals_controller.rb
module Api
  module V1
    class WeeklyGoalsController < BaseController
      def index
        goals = current_user.weekly_goals.order(week_start: :desc)
        render json: goals
      end

      def current
        goal = find_or_initialize_current_goal
        render json: goal
      end

      def update_current
        goal = current_user.weekly_goals.find_or_initialize_by(
          week_start: WeeklyGoal.week_start_for(Date.today)
        )
        if goal.update(weekly_goal_params)
          render json: goal
        else
          render json: { errors: goal.errors.full_messages }, status: :unprocessable_entity
        end
      end

      private

      def find_or_initialize_current_goal
        week_start = WeeklyGoal.week_start_for(Date.today)
        current_user.weekly_goals.find_or_initialize_by(week_start: week_start)
      end

      def weekly_goal_params
        params.require(:weekly_goal).permit(
          :target_hours, :target_questions, :target_percentage
        )
      end
    end
  end
end
```

### UserConfigsController

```ruby
# app/controllers/api/v1/user_configs_controller.rb
module Api
  module V1
    class UserConfigsController < BaseController
      def show
        config = current_user.user_config || current_user.create_user_config!(
          available_days: UserConfig::VALID_DAYS.first(5),
          schedule_strategy: 'sm2'
        )
        render json: config
      end

      def update
        config = current_user.user_config || current_user.build_user_config
        if config.update(user_config_params)
          render json: config
        else
          render json: { errors: config.errors.full_messages }, status: :unprocessable_entity
        end
      end

      private

      def user_config_params
        params.require(:user_config).permit(
          :schedule_strategy,
          available_days: []
        )
      end
    end
  end
end
```

### Specs

```ruby
# spec/requests/api/v1/weekly_goals_spec.rb
RSpec.describe 'Api::V1::WeeklyGoals', type: :request do
  let(:user) { create(:user) }
  let(:headers) { { 'Authorization' => "Bearer #{user_token(user)}" } }

  describe 'GET /api/v1/weekly_goals/current' do
    it 'retorna meta da semana atual (cria se não existir)' do
      get '/api/v1/weekly_goals/current', headers: headers
      expect(response).to have_http_status(:ok)
      body = JSON.parse(response.body)
      expect(body['week_start']).to eq(Date.today.beginning_of_week(:monday).to_s)
    end
  end

  describe 'PUT /api/v1/weekly_goals/current' do
    let(:params) { { weekly_goal: { target_hours: 10, target_questions: 50, target_percentage: 75 } } }

    it 'cria meta da semana' do
      put '/api/v1/weekly_goals/current', params: params, headers: headers, as: :json
      expect(response).to have_http_status(:ok)
      expect(JSON.parse(response.body)['target_hours']).to eq(10.0)
    end

    it 'atualiza meta existente (upsert)' do
      put '/api/v1/weekly_goals/current', params: params, headers: headers, as: :json
      put '/api/v1/weekly_goals/current',
          params: { weekly_goal: { target_hours: 12 } },
          headers: headers, as: :json
      expect(JSON.parse(response.body)['target_hours']).to eq(12.0)
    end

    it 'retorna erro com target_percentage > 100' do
      put '/api/v1/weekly_goals/current',
          params: { weekly_goal: { target_percentage: 110 } },
          headers: headers, as: :json
      expect(response).to have_http_status(:unprocessable_entity)
    end
  end
end

# spec/requests/api/v1/user_configs_spec.rb
RSpec.describe 'Api::V1::UserConfigs', type: :request do
  let(:user) { create(:user) }
  let(:headers) { { 'Authorization' => "Bearer #{user_token(user)}" } }

  describe 'GET /api/v1/user_config' do
    it 'retorna config com defaults se não existir' do
      get '/api/v1/user_config', headers: headers
      body = JSON.parse(response.body)
```
