# Story 8.1 — Backend + Frontend — Paginação na API

## Status: Ready for Review

## Story

**Como** usuário do EduTrack,
**Quero** que as listagens de sessões e temas carreguem de forma paginada,
**Para que** o app permaneça rápido mesmo com grandes volumes de dados históricos.

---

## Contexto

Atualmente todos os endpoints de listagem retornam todos os registros sem limite. Com histórico crescente de `study_sessions` (potencialmente centenas por usuário), isso degradará performance. Esta story implementa paginação server-side com `pagy` (gem leve, zero dependências) nos endpoints mais críticos e adapta o frontend para consumir os metadados de paginação.

**Endpoints prioritários para paginar:**
- `GET /api/v1/study_sessions` — histórico de sessões (cresce indefinidamente)
- `GET /api/v1/subjects/:id/topics` — pode crescer muito em matérias densas

**Endpoints que NÃO precisam de paginação agora:**
- `GET /api/v1/subjects` — tipicamente < 20 por usuário
- `GET /api/v1/weekly_goals` — pequeno volume
- `GET /api/v1/metrics/*` — agregados, não listas

## Dev Notes

### Formato de resposta paginada

```json
{
  "data": [...],
  "meta": {
    "page": 1,
    "per_page": 20,
    "total": 150,
    "total_pages": 8
  }
}
```

### pagy_metadata helper (backend)

```ruby
# app/controllers/api/v1/study_sessions_controller.rb
def index
  sessions = current_user.study_sessions
                         .includes(topic: :subject)
                         .order(scheduled_date: :desc)
  @pagy, @sessions = pagy(sessions, per_page: (params[:per_page] || 20).to_i.clamp(1, 100))
  render json: {
    data: @sessions.as_json(include: { topic: { include: :subject } }),
    meta: pagy_metadata(@pagy)
  }
end
```

### Interface TypeScript (frontend)

```typescript
// frontend/src/app/core/models/paginated-response.model.ts
export interface PaginaMeta {
  page: number;
  perPage: number;
  total: number;
  totalPages: number;
}

export interface PaginatedResponse<T> {
  data: T[];
  meta: PaginaMeta;
}
```

### Gemfile

```ruby
gem 'pagy', '~> 9.0'
```

### Notas técnicas
- `pagy` é mais leve que `kaminari` (zero dependências, ~100x mais rápido para contagem)
- Clampar `per_page` entre 1–100 para evitar abuse
- O frontend consome apenas `data` por enquanto — `meta` é ignorado até implementar scroll infinito
